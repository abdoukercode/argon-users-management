Test pour développeur Front-end

Technologie : Javascript, CSS.

Objectif : L'objectif de cet exercice est d'évaluer votre capacité à organiser votre code front-end en réalisant une interface qui questionne une API donnée.

Exercice : L'interface de gestion des droits.

Partie 1) : Cet exercice est complètement orienté front-end. Il n'est donc pas recquis de fournir du code PHP associé. Vous pouvez néanmoins en faire si vous pensez que c'est nécessaire. À la place du code back-end, nous vous fournissons une API qui vous procurera les données nécessaires au développement de cette interface utilisateur.

Vous trouverez cette API ici : https://abdoutestapi.aragon-erh.com Vous trouverez sa documentation ici : https://abdoutestapi.aragon-erh.com/doc/

Vous devez coder une interface qui permet de gérer les rôles et les droits associés aux utilisateurs d'une application. En utilisant l'API fournie, qui expose les ressources nécessaires, L'interface doit permettre :

- De lister les utilisateurs, les rôles et les droits

- D'ajouter des utilisateurs à des roles

- D'ajouter des droits à un rôle La façon dont les informations sont disposées est laissée à votre libre choix.

Vous avez notamment la possibilité de faire autant de "vues" que vous le souhaitez.

On appréciera, par exemple, la présence des fonctionnalités suivantes :

- Drag'n'Drop

- Gestion et présentation des erreurs remontées par l'API - ...

 

Contraintes technologiques : - Il est interdit d'utiliser des frameworks JS (Angular, React, Vue js, Bootstrap etc...) - En revanche, vous avez le droit d'utiliser des librairies JS et css (Jquery, SASS, Webpack, etc...).

Partie 2) Pour chaque librairie utilisée, Justifiez votre choix en écrivant un petit paragraphe exposant les pours et les contres de l'usage de la techno en question. (Maintenabilité, facilité de développement, performance, etc...) De même, si vous choisissez d'éviter certaines technos en particulier, justifiez votre choix de la même façon.

Enfin, si vous choisissez de tout faire en Javascript et CSS Natif, vous devrez également justifier votre choix.

 

Livraison : Une archive (.zip, .rar, ...) contenant votre code. Ajoutez-y les éléments que vous souhaitez afin que le tout soit facilement compréhensible. Aussi, dans votre mail de réponse, veuillez nous indiquer combien de temps vous avez passé sur cet exercice ainsi que des remarques, si vous en avez.
